# 用户手册：企业级电子文档管理系统 (EDMS)

## 1. 简介

欢迎使用企业级电子文档管理系统 (EDMS)。本系统旨在帮助您在一个集中、透明的环境中创建、编辑、协作审查以及审批内部文档（如规章制度、操作说明、合同等）。

## 2. 快速入门

### 2.1 更新主数据（仅限管理员）

在正式使用之前，系统管理员必须上传员工和部门组织架构。

1. 导航至 **管理控制台 (Admin Dashboard)** (`/admin`)。
2. 上传当前企业的参考数据（`.XLSX` 格式）。
3. 系统将自动解析 Excel 文件，清除旧记录，并使用最新的员工、管理人员和部门信息填充数据库。

### 2.2 系统登录

1. 在网页浏览器中打开应用程序。

2. 输入企业目录中为您分配的授权 **登录名 (Login ID)**。

3. 点击 **登录 (Login)** 按钮。 *(注：根据当前企业配置，无需输入密码)*。

4. 您可以使用顶部导航栏的语言切换器 (Locale Switcher) 切换用户界面语言（英语、俄语、中文）。

   ![CH-FIRST](E:\English Encoding\competition\比赛文件\GUIDE\pic\CH-FIRST.png)

## 3. 个人文档库

登录后，您将被重定向至 **仪表盘 / 个人视图 (Dashboard / Personal View)**。

- 在这里，您可以查看您拥有的所有文档库。
- 您可以根据文档的当前状态轻松筛选文档：**草稿 (Draft)**、**审批中 (In Approval)**、**已批准 (Approved)** 或 **已拒绝 (Rejected)**。

## 4. 文档操作

### 4.1 创建与导入

- **创建：** 点击创建新文档按钮。编辑器中将打开一个空白画布。
- **导入 DOCX：** 使用导入视图上传现有的 Microsoft Word `.docx` 文件。我们的 Python 后端将解析该文件，并在 Web 编辑器中保留其结构和内容。

### 4.2 富文本编辑器功能

系统采用强大的内置富文本编辑器 (Tiptap)。您的修改会自动保存。

- **排版与样式：** 字体大小、加粗、斜体、下划线和删除线。
- **结构：** 多级标题、项目符号列表和编号列表。
- **格式：** 文本对齐、行距、文本颜色和背景高亮。
- **页面设置：** 您可以在编辑器视图中无缝调整排版选项。

### 4.3 导出文档

在任何时候，文档所有者或具有只读权限的用户都可以导出文档。

- 点击编辑器中的 **导出 (Export)** 按钮。

- 选择 **PDF**（通过后端 PDF 工具生成）或 **DOCX** 格式。

- 文件将自动下载到您的本地设备。

## 5. 协同工作与评论

### 5.1 共享访问权限

作为文档所有者，您可以在正式审批之前将草稿共享给同事进行协作。

1. 打开文档并点击 **共享 (Share)**。
2. 将出现一个对话框。选择企业用户并分配权限：**查看 (View)**、**评论 (Comment)** 或 **编辑 (Edit)**。

### 5.2 实时协作

- 当多个拥有“编辑”权限的用户同时打开文档时，系统将通过 WebSockets 建立连接。

- 您将看到彩色的光标，标识您的同事当前正在输入的位置。

- 更改会实时显示在您的屏幕上。

  ![CH-5](E:\English Encoding\competition\比赛文件\GUIDE\pic\CH-5.png)

### 5.3 评论功能

1. 选中文档中的任何文本片段。

2. 添加评论，解释您的建议或问题。

3. 其他用户可以回复您的评论，从而创建一个讨论贴。

4. 一旦问题得到解决，文档所有者或授权编辑可以将该评论标记为 **“已解决 (Completed)”**。

   

## 6. 审批流程

### 6.1 发起审批

当您的草稿定稿后，您可以将其发送以进行正式审批。

1. 点击 **发起审批 (Launch Approval)**。

2. 从企业列表中选择所需的管理人员/审批人。

3. 选择审批类型：
   - **无严格顺序 (No Strict Order)：** 所有指定的审批人可以同时审查并投票。
   - **严格顺序 (Strict Order)：** 审批人必须按规定的顺序依次审查文档。

4. 文档状态将更新为 **审批中 (In Approval)**。

   ![CH-5](E:\English Encoding\competition\比赛文件\GUIDE\pic\CH-6-1.png)

### 6.2 收件箱（针对审批人）

如果您被指派审批某份文档，您将在您的 **收件箱视图 (Inbox View)** 中找到它。

1. 打开文档以审查内容和任何未解决的评论。

2. 点击 **批准 (Approve)** 或 **拒绝 (Reject)**。

3. *如果拒绝：* 您必须输入拒绝的原因。这将被保存到审批历史记录中。文档状态将变为 **已拒绝 (Rejected)**，且编辑功能将被锁定。

   ![CH-6-2](E:\English Encoding\competition\比赛文件\GUIDE\pic\CH-6-2.png)

## 7. 版本控制与差异比对

如果文档被拒绝，所有者必须创建一个新版本来解决管理人员的反馈。

1. 作者编辑文档，创建一个新的迭代版本。

2. 作者重新发起审批流程。

3. 当管理人员审查更新后的文档时，他们可以打开 **差异视图 (Diff View)**（比较版本）。

4. 系统会自动比较旧文本与新文本。它以视觉方式高亮显示 **新增的文本（绿色）** 和 **删除的文本（红色）**，使验证修改内容变得异常简单。

5. 一旦所有审批人接受了更改，文档将被标记为 **已批准 (Approved)**，并成为永久企业文档库的一部分。
