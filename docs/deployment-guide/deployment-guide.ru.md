## Системные требования

### Требования к оборудованию

| Компонент | Минимум | Рекомендуется |
|-----------|---------|---------------|
| CPU | 2 ядра | 4 ядра |
| Память | 2 ГБ | 4 ГБ |
| Дисковое пространство | 5 ГБ | 20 ГБ |
| Сеть | 100 Мбит/с | 1 Гбит/с |

### Требования к программному обеспечению

- Docker 20.10+
- Docker Compose 2.0+

### Поддерживаемые операционные системы

- Основные системы Windows и Linux
- Все системы архитектуры X86, на которых можно корректно установить Docker

## Шаги развертывания

### Шаг 1: Распаковка пакета развертывания

1. **Загрузите пакет развертывания** (`edms-deployment.zip`)
2. **Распакуйте в подходящую директорию**:
   - Windows: Щелкните правой кнопкой мыши и выберите "Извлечь все"
   - Linux/macOS: Используйте `unzip edms-deployment.zip`

### Шаг 2: Установка Docker

- **Windows**: Установите Docker Desktop с официального сайта
- **Linux**: Следуйте официальному руководству по установке Docker для вашего дистрибутива
- **Проверка установки**:
  ```bash
  docker --version
  docker-compose --version
  ```

### Шаг 3: Запуск сервисов

#### Windows

1. **Откройте командную строку** от имени администратора
2. **Перейдите в директорию bin**:
   ```cmd
   cd path\to\edms-deployment\bin\Windows
   ```
3. **Запустите сервисы**:
   ```cmd
   start.bat
   ```

#### Linux/macOS

1. **Откройте терминал**
2. **Перейдите в директорию bin**:
   ```bash
   cd path/to/edms-deployment/bin/Linux
   ```
3. **Добавьте права на выполнение**:
   ```bash
   chmod +x Linux/*.sh
   ```
4. **Запустите сервисы**:
   ```bash
   ./start.sh
   ```

### Шаг 4: Остановка сервисов

#### Windows

```cmd
stop.bat
```

#### Linux/macOS

```bash
./stop.sh
```

## Руководство по настройке

### Переменные окружения

Вы можете настроить развертывание, редактируя файл `bin/.env`:

| Переменная | Описание | По умолчанию | Обязательно |
|------------|----------|--------------|-------------|
| `WEB_PORT` | Порт веб-сервиса | 80 | Да |
| `JWT_SECRET_KEY` | Ключ подписи JWT | Авто-генерация | Да |
| `CORS_ORIGINS` | Разрешенные источники CORS | http://localhost | Нет |
| `DATABASE_URL` | Строка подключения к базе данных | sqlite:///app/data/edms.db | Нет |

**Пример конфигурации**:
```env
# Пользовательский веб-порт
WEB_PORT=8080

# Пользовательский секретный ключ JWT (рекомендуется для продакшена)
JWT_SECRET_KEY=your-secure-secret-key-here

# Разрешить несколько источников
CORS_ORIGINS=http://localhost,http://your-domain.com
```

### Постоянство данных

Все постоянные данные хранятся в директории `bin/data/`:
- `bin/data/backend/` - Данные приложения и база данных SQLite

**Важно**: Регулярно создавайте резервные копии этой директории для предотвращения потери данных.

## Проверка после развертывания

1. **Проверка статуса сервисов**
   ```bash
   docker ps
   ```

2. **Проверка логов**
   ```bash
   # Windows
   docker-compose -f bin\docker-compose.yml logs -f
   
   # Linux/macOS
   docker compose -f bin/docker-compose.yml logs -f
   ```

3. **Тест доступа к фронтенду**
   - Откройте браузер и перейдите по адресу http://localhost
   - Проверьте, что страница входа загружается корректно

## Устранение неполадок

### Порт уже используется

**Ошибка**: `Port 80 is already in use`

**Решение**:
1. Отредактируйте файл `bin/.env`
2. Измените `WEB_PORT` на доступный порт (например, 8080)
3. Перезапустите сервисы

### Контейнер не запускается

**Проверьте логи**:
```bash
docker logs edms-backend
docker logs edms-frontend
```

**Частые причины**:
- Недостаточно дискового пространства
- Ограничения памяти
- Конфликты портов

### Проблемы подключения к базе данных

**Решение**:
1. Проверьте права доступа к директории данных
2. Проверьте целостность файла базы данных
3. Просмотрите логи бэкенда для конкретных ошибок

## Операции обслуживания

### Просмотр логов

```bash
# Windows
docker-compose -f bin\docker-compose.yml logs -f

# Linux/macOS
docker compose -f bin/docker-compose.yml logs -f
```

### Перезапуск сервисов

```bash
# Windows
docker-compose -f bin\docker-compose.yml restart

# Linux/macOS
docker compose -f bin/docker-compose.yml restart
```

### Резервное копирование данных

```bash
# Сначала остановите сервисы
./Linux/stop.sh  # или Windows\stop.bat

# Создайте резервную копию директории данных
tar -czvf edms-backup-$(date +%Y%m%d).tar.gz bin/data/

# Перезапустите сервисы
./Linux/start.sh  # или Windows\start.bat
```

### Обновление развертывания

1. **Создайте резервную копию существующих данных**
2. **Остановите сервисы**
3. **Распакуйте новую версию поверх существующих файлов**
4. **Запустите сервисы**
